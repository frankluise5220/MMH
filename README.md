# MMH

MMH 是一个面向 NAS / 本地 Docker 自托管的家庭财务系统，提供 Web 工作台和移动端客户端。

它不是把家庭账本、资产和投资数据交给外部平台托管的 SaaS，而是把数据留在你自己的设备里。

## 为什么做 MMH

家庭财务数据天然敏感，往往包括：

- 账户余额和交易流水
- 基金持仓和收益变化
- 借贷、保险和现金流
- 家庭成员与账户归属
- 邮箱账单、账单解析和后续统计

这些数据一旦分散在多个银行、基金平台、账单邮件和不同设备里，后期就会出现几个很典型的问题：

- 数据很敏感，但又不想交给外部云平台
- 多账户、多资产分散，长期很难统一管理
- 日常录入和复杂统计常常只能二选一
- 投资数据不能只停留在“记了一笔”

MMH 想解决的，就是把这些数据统一到一个可长期使用的家庭财务底座里。

## 项目特点

- 自托管优先：适合 NAS / 本地 Docker 部署，数据自己掌控
- 统一视图：把账户、流水、基金、借贷、保险放到同一套账簿体系里
- Web + 移动端：Web 做细致工作台，移动端做日常查看和快速录入
- 统一口径：同一套后端数据语义和计算规则，减少不同页面、不同端结果不一致
- 面向投资：基金交易、持仓、净值、收益、到账日期不是零散拼接，而是统一处理
- 可继续扩展：定投、批量记账、账单解析、AI 识别、多客户端 API 会持续完善

## 适合什么场景

- 想把家庭财务数据放在自己设备里
- 不满足于普通记账 App，只想看更完整的资产和投资视图
- 有多张卡、多账户、多平台资产，需要统一统计
- 希望 Web 和移动端共享同一套数据和规则
